Final Year Projects

Privacy Preserving Analytics

As organizations increasingly collect sensitive information about individuals, these organizations are ethically and legally obligated to safeguard against privacy leaks. We also need to allow some collaborative analysis along with preserving privacy. Our project intends to solve this tradeoff without modifying the databases, thereby making it database independent.

Our approach must support heterogeneous database environments by not requiring a specific database distribution or replacement of the database with a custom runtime. We mainly target five standard SQL aggregation functions: Count, Sum, Average, Max, and Min, which represent 99% of aggregation functions used in real-world queries.

Automatic Bug Repair

MOOCs offer large scale programming education, through introductory courses. Evaluating students assignments and providing them with helpful feedback is a laborious task. Automating the process of providing feedback to students would help make MOOCs scalable. This would mean correcting incorrect student submissions and converting that correction into a natural language format.

Many tools exist to make corrections to incorrect programs. But, none of them provide good feedback to the student about their conceptual misunderstandings. The student must make do with a message similar to a compiler error message. This is an area that we find is seriously lacking and would like to address through our project of automatic bug repair and feedback.

Concurrency Bug Detection

Breaking down programs into tasks is a common practice while programming. A project may contain a large number of tasks that have to be run. These tasks if run serially may not be able to perform up to their full potential. Hence, parallel execution of tasks has become an approach many programmers have adopted to speedup programs and achieve greater results.

Parallelizing code could lead to a large number of timing problems while sharing data and accessing resources. In this project, we explore different types of concurrency bugs and propose an efficient concurrency bug detector to improve performance of code and to speed up the bug detection process.

Programming Frameworks for Heterogeneous Computing

As there is high demand for GPGPUs in market, so its framework. CUDA which is proprietary to Nvidia runs better on its own platform. And on the other hand OpenCL is a heterogenous framework which runs over all types of platform CPUs,GPUs,FPGAs,Accelerators. There are developers or vendors who want to translate code from CUDA to OpenCL to develop CUDA code in OpenCL or who want to check their code in different platforms.

CU2CL reduces the time and cost of manual translation of code. The main aim of project is to analyse the efficiency of existing CU2CL.

Patents


Tolerating and detecting asymmetric races


Detecting and/or tolerating races. Races occur due to malicious threads not respecting software locks. A method of detecting and/or correcting races includes making local copie(s) and reference copie(s) of shared data. Any read and write operations performed by a safe thread are caused to be performed on the local copie(s) during a critical section. The critical section defines a time frame which a variable lock is placed on shared data. Any read and write operations performed by malicious threads are allowed to be performed on the shared data during the critical section. The shared data, the local copie(s), and the reference copie(s) are compared to determine that a race has been detected. An indication can be output that a race has occurred or the race can be corrected. Read More


Method and system for maintaining vector clocks during synchronization for data race detection


Method and system for maintaining vector clocks during synchronization for data race detection. Embodiments herein disclose methods to reduce overheads of maintaining and updating vector clock during synchronization in vector based dynamic data race detection systems. Embodiments herein enable improvement of vector based dynamic data race detection systems orthogonally without compromising with precision of the system by using opportunistic methods to reduce overheads during synchronization of threads.
